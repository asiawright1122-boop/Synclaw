# SynClaw 全平台产品设计总览

> 落地页 · 桌面端 · 客户 Portal · 平台后台 · 商业化 · IM 渠道

---

## 1. 产品愿景

**SynClaw** — 让 AI 成为你的桌面伙伴。

数据永不离开你的电脑，AI 随时在你的 IM 中响应。

## 3. 核心架构

### 3.1 整体架构

```
┌─────────────────────────────────────────────────────────────────────────┐
│                              用户层                                        │
│                                                                          │
│  ┌──────────────────────┐  ┌──────────────────────┐  ┌──────────────┐ │
│  │       桌面端          │  │       Web 端          │  │   移动端     │ │
│  │      Electron         │  │     (Web App)         │  │ (React Nat.) │ │
│  │                       │  │                       │  │              │ │
│  │  OpenClaw Gateway    │  │  连接桌面 Gateway     │  │ 连接桌面    │ │
│  │  (本地运行)          │  │  (需要桌面端在线)    │  │ Gateway    │ │
│  └──────────────────────┘  └──────────────────────┘  └──────────────┘ │
│                                                                          │
└─────────────────────────────────────────────────────────────────────────┘
                                    │ HTTPS / WebSocket
                                    ▼
┌─────────────────────────────────────────────────────────────────────────┐
│                            SynClaw Cloud                                  │
│                                                                          │
│  ┌─────────────────┐  ┌─────────────────┐  ┌─────────────────────────┐ │
│  │   API Gateway   │  │  WebSocket      │  │    SynClaw Services    │ │
│  │   (认证/路由)   │  │   Server        │  │                         │ │
│  │                 │  │                 │  │  • 用户 & 订阅管理      │ │
│  │  • JWT 认证     │  │  • 实时通信     │  │  • 积分系统            │ │
│  │  • 限流         │  │  • 推送通知     │  │  • 支付处理            │ │
│  │  • 路由分发     │  │  • 桌面唤醒     │  │  • IM 消息路由         │ │
│  └─────────────────┘  └─────────────────┘  └─────────────────────────┘ │
│                                                                          │
│  ┌─────────────────┐  ┌─────────────────┐  ┌─────────────────────────┐ │
│  │   PostgreSQL    │  │     Redis       │  │       支付服务          │ │
│  │   (主数据库)    │  │   (缓存/队列)   │  │                         │ │
│  │                 │  │                 │  │  • Stripe (海外)        │ │
│  │  • 用户数据     │  │  • Session      │  │  • 微信/支付宝 (国内)   │ │
│  │  • 订阅记录     │  │  • 限流计数     │  │                         │ │
│  │  • 积分流水     │  │  • 消息队列     │  │                         │ │
│  │  • 订单数据     │  │                 │  │                         │ │
│  └─────────────────┘  └─────────────────┘  └─────────────────────────┘ │
│                                                                          │
└─────────────────────────────────────────────────────────────────────────┘
```

### 3.2 数据流

```
用户操作 (IM / 桌面)
        ↓
Cloud API (认证 / 限流)
        ↓
OpenClaw Gateway (AI 处理)
        ↓
结果返回 / IM 回复
```

## 6. 技术栈汇总

| 层级 | 技术 |
|------|------|
| **前端框架** | Next.js 15 (App Router) |
| **桌面端** | Electron 28 + React 18 + TypeScript |
| **样式** | Tailwind CSS v4 |
| **组件库** | shadcn/ui + Radix |
| **动画** | Framer Motion |
| **状态管理** | Zustand (UI) + React Query (Server) |
| **后端** | Node.js (Next.js API) + OpenClaw Gateway |
| **数据库** | PostgreSQL |
| **缓存** | Redis |
| **部署** | Vercel (Web) + 自行部署 (Gateway) |
| **支付** | Stripe + 微信支付 + 支付宝 |
| **IM 适配** | 各平台 SDK |

## 7. 分阶段路线图

### Phase 0: 核心打磨（当前）

**目标**: 稳定桌面端核心功能

```
✅ 已完成:
  • Electron 桌面客户端基础框架
  • OpenClaw Gateway 集成
  • AI 对话基本功能
  • 本地文件操作

🔄 进行中:
  • 技能系统完善
  • UI 优化打磨
  • 稳定性提升
```

### Phase 1: 产品基础设施

**时间**: 1-2 个月

```
交付物:
├── 🌐 落地页上线
│   └── 品牌展示、下载入口、定价页
│
├── 🔐 客户 Portal v1
│   └── 用户注册/登录、订阅管理、积分基础
│
├── 📊 平台后台 v1
│   └── 客户管理、订单查看、基础统计
│
└── 💳 支付系统 v1 (海外)
    └── Stripe 订阅 + 积分充值
```

### Phase 2: 商业化启动

**时间**: 2-3 个月

```
交付物:
├── 💰 国内支付接入
│   └── 微信支付 + 支付宝
│
├── 📱 IM 渠道 v1
│   └── 飞书 Bot + 企业微信
│
├── 📈 客户 Portal 完善
│   └── API Key、使用统计、IM 配置
│
└── 📊 平台后台完善
    └── 工单系统、IM 监控
```

### Phase 3: 功能扩展

**时间**: 3-6 个月

```
交付物:
├── 🌏 更多 IM 渠道
│   └── 钉钉 → Telegram → Slack/Discord

### Phase 4: 企业级

**时间**: 6 个月+

```
交付物:

│   └── 私有化部署、专属支持
