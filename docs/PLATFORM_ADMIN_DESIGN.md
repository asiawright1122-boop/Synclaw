# SynClaw 平台管理后台设计文档

> admin.synclaw.ai (或内部域名) — SynClaw 运营方内部使用

---

## 1. 产品定位

| 维度 | 内容 |
|------|------|
| **产品名称** | SynClaw Platform Admin |
| **访问地址** | `admin.synclaw.ai`（仅内部访问） |
| **目标用户** | SynClaw 运营团队（内部） |
| **核心定位** | 运营方管理所有客户：租户、订单、套餐、工单、数据 |
| **设计风格** | 深色主题，参考 Stripe Dashboard、Linear，功能导向，信息密集 |
| **访问控制** | 仅内部网络 / VPN + 邮箱白名单 |

---

## 2. 与客户 Portal 的区分

| 对比维度 | 客户 Portal | 平台管理后台 |
|----------|-------------|-------------|
| **受众** | C 端用户 | 运营团队（内部） |
| **功能** | 自助管理 | 全局管控 |
| **数据范围** | 自己的数据 | 所有客户数据 |
| **敏感操作** | 受限 | 全部权限 |
| **设计风格** | 简洁友好 | 功能密集、表格为主 |

---

| 角色 | 说明 | 权限 |
|------|------|------|
| **超级管理员** | 全部权限 | 所有操作 + 系统配置 |

| **财务** | 财务相关 | 订单、收入统计（不含客户敏感信息） |

## 7. 技术实现

### 7.1 技术选型

| 技术 | 选择 | 理由 |
|------|------|------|
| **框架** | Next.js 15 (App Router) | 与落地页、Portal 共享代码库 |
| **UI 组件** | shadcn/ui + Radix | Accessible，表格组件强大 |
| **表格** | TanStack Table | 功能最全的 React 表格库 |
| **图表** | Recharts | 轻量，与 React 集成好 |
| **状态** | React Query + Zustand | 数据获取 + UI 状态 |
| **权限** | NextAuth.js + RBAC | 细粒度权限控制 |
| **数据库** | PostgreSQL (已有) | 多租户设计 |

### 7.2 API 设计

| 端点 | 方法 | 说明 |
|------|------|------|
| `/api/admin/customers` | GET | 客户列表（分页、筛选） |
| `/api/admin/customers/:id` | GET/PATCH | 客户详情、编辑 |
| `/api/admin/orders` | GET | 订单列表 |
| `/api/admin/subscriptions` | GET/PATCH | 订阅管理 |
| `/api/admin/packages` | GET/POST/PATCH | 套餐 CRUD |
| `/api/admin/tickets` | GET/POST/PATCH | 工单 CRUD |
| `/api/admin/stats` | GET | 数据统计 |
| `/api/admin/settings` | GET/PATCH | 系统配置 |
| `/api/admin/channels` | GET/PATCH | IM 渠道配置 |
| `/api/admin/logs` | GET | 操作日志 |
