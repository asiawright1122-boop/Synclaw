## 一、产品定位

OpenClaw 是一个 SaaS 化的本地 AI Agent 平台，核心价值：

> **让用户的 AI Agent 安全地访问本地资源，完成各种自动化任务**

### 核心理念

- **数据不落地** - 敏感文件只在本地处理，云端不存储
- **能力可扩展** - 通过 ClawHub 安装技能
- **随时可触发** - 支持 IM 双向交互

## 四、ClawHub 集成设计

### 4.1 技能类型

| 类型 | 执行位置 | 示例 |
|------|----------|------|
| **local** | 本地执行 | 文件监控、Shell 执行 |
| **cloud** | 云端执行 | 代码分析、知识库问答 |
| **hybrid** | 混合执行 | 本地读取 + 云端分析 |

### 4.2 技能结构

```yaml
skill:
  id: "skill-001"
  name: "代码分析"
  type: "hybrid"           # 混合执行
  category: "development"
  description: "分析代码质量并提供改进建议"
  version: "1.0.0"
  author: "OpenClaw"
  
  # 触发方式
  triggers:
    - type: "command"     # 命令触发
      commands: ["分析", "analyze"]
    - type: "im"         # IM 触发
      platform: "feishu"
  
  # 执行定义
  execution:
    - step: "read_file"
      type: "local"       # 本地读取
      action: "file_read"
    - step: "analyze"
      type: "cloud"      # 云端分析
      agent: "code-review-agent"
  
  # 配置参数
  config:
    - name: "include_patterns"
      type: "array"
      default: ["*.go", "*.ts", "*.js"]
```

### 4.3 安装流程

```
用户点击安装
    │
    ▼
客户端 → Gateway: POST /api/v1/skills/:id/install
    │
    ▼
Gateway → ClawHub: 获取技能定义
    │
    ▼
Gateway: 存储用户技能配置
    │
    ▼
客户端: 本地缓存技能配置
    │
    ▼
用户可以在本地使用此技能
```

## 五、IM 双向集成

### 5.1 支持的渠道

| 渠道 | 方向 | 优先级 |
|------|------|--------|
| 飞书 | 收发双向 | P0 |
| Slack | 收发双向 | P0 |
| Discord | 收发双向 | P1 |
| Telegram | 收发双向 | P2 |

### 5.2 交互流程

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                         IM 双向交互                                        │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  1. 用户 → OpenClaw (触发任务)                                             │
│     用户在 IM 发送: "帮我分析 /path/to/file.go"                            │
│           │                                                                  │
│           ▼                                                                  │
│     IM Webhook → Gateway → 解析命令 → 创建任务 → 执行                       │
│                                                                             │
│  2. OpenClaw → 用户 (推送结果)                                             │
│     任务完成 → 格式化结果 → IM Webhook → 用户                              │
│                                                                             │
│  3. 用户 → OpenClaw (查询状态)                                             │
│     用户在 IM 发送: "查看用量"                                             │
│           │                                                                  │
│           ▼                                                                  │
│     IM Webhook → Gateway → 查询 → 返回结果                                 │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

### 5.3 支持的命令

| 命令 | 描述 | 示例 |
|------|------|------|
| `分析 <路径>` | 分析指定文件 | "分析 /Users/kaka/main.go" |
| `监控 <路径>` | 启动目录监控 | "监控 /Users/kaka/docs" |
| `查看用量` | 查看用量统计 | "查看用量" |
| `任务列表` | 列出运行中的任务 | "任务列表" |
| `帮助` | 显示帮助信息 | "帮助" |

## 六、数据流设计

### 6.1 任务执行流程

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                        任务执行流程                                        │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  1. 用户触发 (客户端 / IM)                                                 │
│           │                                                                  │
│           ▼                                                                  │
│  2. 客户端 → Gateway: 创建任务请求                                          │
│           │                                                                  │
│           ▼                                                                  │
│  3. Gateway: 验证权限 + 记录任务                                           │
│           │                                                                  │
│           ▼                                                                  │
│  4. 任务调度 (根据 SKILL 定义)                                             │
│           │                                                                  │
│     ┌────┴────┐                                                            │
│     ▼         ▼                                                             │
│  本地执行   云端执行                                                        │
│     │         │                                                             │
│     ▼         ▼                                                             │
│  文件操作   LLM 调用                                                        │
│     │         │                                                             │
│     └────┬────┘                                                            │
│           │                                                                  │
│           ▼                                                                  │
│  5. Gateway: 汇总结果                                                      │
│           │                                                                  │
│           ▼                                                                  │
│  6. 推送结果 (客户端 + IM)                                                 │
│           │                                                                  │
│           ▼                                                                  │
│  7. 用户收到通知                                                            │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

### 6.2 数据安全

| 数据类型 | 存储位置 | 加密 |
|----------|----------|------|
| 用户账号信息 | 云端 | ✅ |
| 订阅信息 | 云端 | ✅ |
| 技能配置 | 云端 + 本地缓存 | ✅ (云端) |
| 任务日志 | 云端 | ✅ |
| 文件内容 | **本地** | - |
| 敏感文件路径 | 本地缓存 | - |
