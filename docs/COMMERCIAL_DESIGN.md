## 3. 积分系统设计

### 3.1 积分作为通用计量单位

```
┌─────────────────────────────────────────────────────────────┐
│                    积分抽象层                                │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│  用户视角: 积分 = 通用消费单位                                │
│  ─────────────────────────────────────────────────────     │
│  • 充值积分 (购买)                                           │
│  • 消耗积分 (使用 AI 功能)                                   │
│  • 查看余额 (余额查询)                                        │
│  • 积分不够? 订阅 Pro 套餐或单独充值                          │
│                                                              │
│  后台视角: 积分映射到各 AI 渠道                               │
│  ─────────────────────────────────────────────────────     │
│  • OpenAI (GPT-4): $0.03/1K Token → 30积分/1K Token         │
│  • Anthropic (Claude): $0.008/1K Token → 8积分/1K Token     │
│  • Claude 3 Opus: $0.015/1K Token → 15积分/1K Token         │
│                                                              │
│  汇率设计                                                    │
│  ─────────────────────────────────────────────────────     │
│  • 国际: 1000积分 = $1.00                                    │
│  • 国内: 1000积分 = ¥7.00 (按汇率 1:7)                      │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

### 3.2 积分计费规则

| AI 操作 | 计费方式 | Pro 套餐覆盖 | 超额费率 |
|---------|---------|-------------|---------|
| 文本对话 (Claude) | 积分/1K Token | ✅ 含 1000积分/月 | +8积分/1K |
| 文本对话 (GPT-4) | 积分/1K Token | ✅ 含 1000积分/月 | +30积分/1K |
| 图片理解 | 积分/张 | ❌ 全额计费 | 50积分/张 |
| 技能执行 | 积分/次 | ❌ 全额计费 | 20-200积分/次 |
| 文件操作 | 免费 | ✅ | 免费 |
| TTS 语音 | 积分/字符 | ❌ 全额计费 | 1积分/100字符 |

### 4.2 订阅支付流程

```
用户订阅 Pro 套餐（以 Stripe 为例）

1. 用户选择套餐
   ┌─────────────────────────────────────────────────────┐
   │  [Pro - $9/月]  [选择支付方式]                       │
   └─────────────────────────────────────────────────────┘
                        ↓
2. 跳转 Stripe Checkout
   ┌─────────────────────────────────────────────────────┐
   │  Stripe 托管支付页面                                  │
   │  - 信用卡输入                                       │
   │  - Apple Pay / Google Pay                           │
   │  - 地址信息                                        │
   └─────────────────────────────────────────────────────┘
                        ↓
3. 支付成功 → Stripe Webhook 回调
   POST /api/webhooks/stripe
   {
     "type": "checkout.session.completed",
     "data": { "customer": "cus_xxx", "subscription": "sub_xxx" }
   }
                        ↓
4. 后端处理
   ┌─────────────────────────────────────────────────────┐
   │  1. 创建/更新用户订阅记录                            │
   │  2. 开通对应功能权限                                 │
   │  3. 发送欢迎邮件                                    │
   │  4. 记录支付流水                                    │
   └─────────────────────────────────────────────────────┘
                        ↓
5. 完成
   ┌─────────────────────────────────────────────────────┐
   │  订阅成功!                                          │
   │  您现在是 Pro 用户                                  │
   └─────────────────────────────────────────────────────┘
```

### 4.4 订阅续费流程

```
自动续费（Stripe / 微信支付自动扣款）

每月 20日 (用户订阅日)
        ↓
1. Stripe 自动扣款
   - 信用卡 / Apple Pay
   OR
   微信支付代扣 (需用户签约)
        ↓
2. 扣款成功
   ┌─────────────────────────────────────────────────────┐
   │  发送收据邮件                                        │
   │  更新订阅状态                                        │
   │  续期服务有效期                                      │
   └─────────────────────────────────────────────────────┘
        ↓
3. 扣款失败
   ┌─────────────────────────────────────────────────────┐
   │  重试 (3次，每次间隔1天)                            │
   │  发送续费失败提醒                                    │
   │  到期后降级为 Free (保留数据7天)                    │
   └─────────────────────────────────────────────────────┘
```

### 4.5 退款流程

```
用户申请退款

1. 用户发起退款
   ┌─────────────────────────────────────────────────────┐
   │  联系客服或客户 Portal 申请退款                      │
   │  退款原因: [订阅未生效 / 重复扣费 / 其他]           │
   └─────────────────────────────────────────────────────┘
                        ↓
2. 客服审核
   - 订阅未生效: 7天内可全额退款
   - 重复扣费: 全额退款
   - 其他: 酌情处理 (按剩余天数比例退款)
                        ↓
3. 执行退款
   订阅退款 → Stripe / 微信支付原路退回
   积分退款 → 兑换比例折算退款
                        ↓
4. 完成
   ┌─────────────────────────────────────────────────────┐
   │  退款处理完成                                       │
   │  - 订阅取消                                        │
   │  - 金额原路退回 (3-7个工作日)                       │
   │  - 发送退款确认邮件                                 │
   └─────────────────────────────────────────────────────┘
```

## 6. 定价策略

### 6.1 定价原则

1. **价值定价** — 定价与用户获得的价值匹配
2. **竞争参考** — 参考 Cursor ($20/月 Pro)、Raycast ($8/月 Pro)
3. **转化优化** — Free → Pro 是核心转化路径
4. **地区差异化** — 国内价格更低（按购买力调整）

### 6.2 竞品对比

| 产品 | Free | Pro | Team |
|------|------|-----|------|
| **SynClaw** | $0 | $9 | $29 |
| Cursor | $0 | $20 | $30 |
| Raycast | $0 | $8 | $16 |
| GitHub Copilot | $0 | $10 | $19 |
| Perplexity | $0 | $20 | $20 |

**SynClaw 定价优势：** 低于 Cursor 50%+，但功能更丰富（IM 渠道、文件操作）

### 6.3 定价心理

| 策略 | 应用 |
|------|------|
| **锚定效应** | 展示 Enterprise 价格 ($999) 让 Pro ($9) 看起来很便宜 |
| **损失厌恶** | "现在升级 Pro，享首月 5 折" |
| **紧迫感** | "限时优惠：年付 8 折，仅剩 X 天" |
| **对比效应** | Pro 卡片加边框 + "最受欢迎" 标签 |

| 项目 | 说明 |
|------|------|

| 退款汇总 | 每日退款金额 |

| 微信/支付宝对账 | 下载商户后台流水对比 |

### 8.2 月对账

| 项目 | 说明 |
|------|------|
| 月度收入报告 | 各产品线收入 |
| 用户订阅变化 | 新增 / 升级 / 降级 / 流失 |
| 毛利率计算 | 收入 - Stripe/微信手续费 - AI API 成本 |
| 佣金结算 | 技能市场分成 (如有) |

### 8.3 关键指标

| 指标 | 计算方式 | 健康值 |
|------|---------|--------|
| MRR | 月度经常性收入 | 持续增长 |
| ARR | MRR × 12 | 年度目标 |
| 流失率 | 流失用户 / 总用户 | < 5%/月 |
| LTV | 用户平均生命周期价值 | > $100 |
| CAC | 获客成本 | < $20 |
| LTV:CAC | LTV / CAC | > 3:1 |
| 付费转化率 | 付费用户 / 总用户 | > 5% |
