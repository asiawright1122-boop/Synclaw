# SynClaw 客户 Portal 设计文档

> portal.synclaw.ai — 用户自管理后台（C 端用户使用）

---

## 1. 产品定位

| 维度 | 内容 |
|------|------|
| **产品名称** | SynClaw Portal |
| **访问地址** | `portal.synclaw.ai` |
| **目标用户** | 所有已注册 C 端用户 |
| **核心定位** | 用户自主管理：订阅、积分、API Key、个人信息 |
| **设计风格** | 与落地页一致的暗色主题，简洁专业，参考 Vercel Dashboard |

## 6. 技术实现

### 6.1 技术选型

| 技术 | 选择 | 理由 |
|------|------|------|
| **框架** | Next.js 15 (App Router) | SSG/SSR，与落地页共享 |
| **UI 组件** | shadcn/ui + Radix | 暗色主题，Accessible |
| **样式** | Tailwind CSS | 原子化，开发效率高 |
| **动画** | Framer Motion | 页面切换动画 |
| **图表** | Recharts | 轻量、React 原生 |
| **表单** | React Hook Form + Zod | 类型安全、表单验证 |
| **状态** | React Query | 数据获取与缓存 |
| **部署** | Vercel | 与落地页共享 |

### 6.2 认证方案

- **Auth**: NextAuth.js (支持邮箱密码、Google OAuth)
- **Session**: JWT，存储在 HTTP-only Cookie
- **API 路由**: `/api/*` 所有路由需要认证

### 6.3 API 设计

| 端点 | 方法 | 说明 |
|------|------|------|
| `/api/user/profile` | GET/PATCH | 个人信息 |
| `/api/subscription` | GET/POST | 订阅信息 |
| `/api/credits` | GET/POST | 积分管理 |
| `/api/credits/history` | GET | 积分流水 |
| `/api/api-keys` | GET/POST/DELETE | API Key CRUD |
| `/api/usage` | GET | 使用统计 |
| `/api/channels` | GET/PATCH | IM 渠道配置 |
| `/api/notifications` | GET/PATCH | 通知偏好 |
