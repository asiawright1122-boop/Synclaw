## Key Technical Decisions

1. **Gateway 代理模式**：`web/` 作为独立 Next.js 应用，通过 server-side HTTP 与 Gateway 通信，不使用 Electron IPC
2. **Avatar 沙箱隔离**：Avatar 切换通过 Gateway sandbox API 管理，不在 client 内直接操作
3. **TTS 同步高亮**：Gateway `tts.convert` 是请求-响应模式，前端用 `audio.duration / wordCount` 估算 per-word 时长
4. **审批增强**：超时不只是拒绝，还附带 reason 字段用于 audit trail
5. **签名架构**：公证通过 `electron-builder` `--publish never` + `node scripts/notarize.js` 实现
